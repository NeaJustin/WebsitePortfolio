CSE 431: Algorithm Analysis

Algorithm Analysis is one that was extensively covered in that it was a lot of understanding heaps and sorting algorithms to imrpove time complexity. One of the main things was understanding why a program would take O(n), n^2, linear time, or it could be O(nlogn). Each one varying based on what the algorithm was, how we found it through proofs or through proof by induction. Also, when going through more proofs and from there finding ways to improve on the algorithm. One example would be merge sort in imporving the time complexity. Or using a sift-down algorithm to sort a binary tree when given an array of numbers being added in random order.

CSE 460: Operating Systems

Github: CSE 460 For CSE 460, we were put into groups and the labs were broken up into three phases. Each phase consisting of different parts that would be used for the project.
It consisted of learning about memory management, time splicing, creating object files for the op codes to run, creating op codes to translate to bits, creating an 3 different registers such as general purpose registers, SR (Status Register), and IR (Instruction Register). Throughout the project, we started implementing more things to help make it simulate an operating system such as paging, multi-threading, scheduling, and more things throughout. Each phase of the project can be found using the github link. Not all was created by me, some of it was provided in the beginning of the project.
